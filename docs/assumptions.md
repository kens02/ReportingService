# 仮定した仕様

- 認証はデモ運用を想定し、NextAuthのCredentials Providerで環境変数のデモユーザーを認証する。
- レポートの可視化データは「ラベルのカンマ区切り」と「数値のカンマ区切り」で入力する。
- レポートデータはSQLiteを利用し、VercelではPrismaのマイグレーションを実行する前提とする。
- レポート作成・更新・削除はログイン済みユーザーのみ可能とする。
- レポート詳細のデータ構造は JSON 文字列として `{"series":[{"label","value"}]}` を `data` に保持する。
- ページネーションは1ページ6件表示とする。
