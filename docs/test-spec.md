# テスト仕様

## テスト戦略
- バリデーション/変換ロジックを単体テスト
- UI は Next.js の統合動作で確認

## 単体テスト観点
- 入力バリデーションが正しく動作する
- ラベルと値の変換が正しく行われる

## 結合テスト観点
- レポート作成 → 一覧表示 → 詳細表示の一連フロー
- 認証あり/なしでのアクセス制御

## 境界値テスト
- タイトル最小文字数 (3)
- 説明最小文字数 (5)
- ラベル/値の件数が1件の場合

## 異常系テスト
- 空の入力でバリデーションエラー
- チャート種別不正
- 未認証での CRUD API へのアクセス

## 負荷テスト方針
- 1ページあたり6件の表示を基本とし、将来的に 1000件程度までの一覧取得を想定
