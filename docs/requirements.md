あなたはシニアフルスタックエンジニアです。

以下の要件に基づき、Vercelにデプロイ可能なWebアプリケーションを設計・実装してください。

# **■ プロジェクト概要**

ReportingServiceという、データを集計・可視化するWebアプリを開発する。

# **■ 技術スタック（指定）**

- フレームワーク: Next.js（App Router）
- 言語: TypeScript
- フロントエンド: React（Server Components優先）
- スタイリング: Tailwind CSS
- UI: shadcn/ui
- データ取得: Server Actions or Route Handlers
- 状態管理: 必要最小限（useState / Server中心）
- 認証: NextAuth.js
- DB: Prisma
- デプロイ: Vercel前提

# **■ 機能要件**

1. レポート一覧表示
    - 保存されたレポートを一覧で表示
    - ページネーション or 無限スクロール対応
2. レポート作成
    - 入力フォームからレポートを作成
    - バリデーションあり
3. レポート詳細表示
    - 個別レポートの詳細確認
    - グラフまたは集計データを表示
4. データ可視化
    - シンプルなグラフ表示（例: 棒グラフ、折れ線）
    - ライブラリは軽量なものを選定（例: Recharts）
5. API設計
    - RESTまたはNext.js Route Handlersで実装
    - CRUD操作を提供

# **■ 非機能要件**

- Vercelでそのまま動作すること
- 環境変数で設定を管理
- 型安全を徹底（TypeScript）
- 可読性の高い構成
- パフォーマンス最適化（Server Components活用）
- SEO考慮（必要なページのみ）

# **■ ディレクトリ構成**

- app/
- components/
- lib/
- hooks/
- prisma/
- styles/

# **■ UI/UX要件**

- ミニマルでモダンなUI
- レスポンシブ対応
- ローディング・エラーハンドリングあり

# **■ 出力してほしい内容**

1. プロジェクト構成（フォルダツリー）
2. 主要ファイルのコード（省略なし）
3. セットアップ手順
4. Vercelデプロイ手順
5. 環境変数一覧
6. 拡張案（今後の改善）

# **■ 制約**

- 実行可能なコードのみ出力
- ダミーデータで動作確認できること
- 外部依存は最小限に
